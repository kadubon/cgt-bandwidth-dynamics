# Changelog

All notable changes will be documented here.

The format follows a lightweight Keep a Changelog style, and the project uses
SemVer once public releases begin.

## 0.1.0 - Unreleased

- Initial finite executable CGT bandwidth reference implementation.
- Added typed residual rows, bounded closure, evidence closure, stage evidence,
  component profiles, support antichains, completion classes, release checking,
  and release-cover synthesis.
- Added conformance-oriented certificates for well-formedness, closure, support,
  completion, and release-action descent.
- Added strict conformance options, checker descriptors, provenance-sensitive
  evidence audit, richer RECH/support/completion/release certificates, and
  `cgt-bw close --certificate`.
- Added finite `StoreStepRule` closure tables, `RuleTableCertificate`,
  component/support/completion law certificates, strict release predicate
  coordinate checks, and support-signature based release descent proof records.
- Added a strict executable spec schema and a strict minimal example for
  cross-language conformance runners.
- Added strict closure rejection for legacy `payload.derives`, accepted/failure/
  boundary/candidate/derived row certificate fields, and a closure-certificate
  JSON Schema.
- Added nine-clause RECH certificate results, strict typed retyping path checks,
  finite graph component evaluators, component debt/support table totality
  checks, and ambient row-universe release validation.
- Added release-descent certificate JSON Schema and stricter release proof-map,
  checker-trace, predicate-coordinate, and same-source-class descent witnesses.
- Added explicit `ModeTableCertificate`, `cgt-bw normalize-strict`, typed RECH
  node/arc output, `StorePreservationCertificate`, `ReadCoordUniverseCertificate`,
  `CompletionUniversalPropertyCertificate`, `ReleaseTransformationCertificate`,
  `CertifiedExactReleaseCertificate`, and `raw_release_closure_exact`.
- Added CI coverage gate at 94%, expanded schema validation for official
  certificate output, publication audit script, and artifact-content scans for
  release hygiene.
- Documented the v0.1.0 scope as finite, certificate-relative executable
  semantics; infinite domains, external abstraction proofs, and proof assistant
  integration remain out of scope.
